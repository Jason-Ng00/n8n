# Developer Notes - 2026-03-24

- Initialized the minimal repository structure for the n8n-python wrapper.
- Built a `Dockerfile` derived from `n8nio/n8n:latest` leveraging `apk add` to install Python dependencies.
- Opted to use `--break-system-packages` flag in `pip` installation due to Alpine 3.19+ preventing global pip installations; this is completely safe in this containerized scenario.
- Abstracted Python library addition into a flexible `requirements.txt` which handles pre-installation cleanly.
- Authored a clean and concise `README.md` containing specific instructions for configuring Render deployments (i.e. Web Service + Git Repo approach, avoiding docker-compose).
- Logged project specifications to `project.md` aligning with user rules.
- **Troubleshooting Update**: The official `n8nio/n8n:latest` Docker image is highly stripped down (distroless) and completely removes its native package managers (`apk`, `apt-get`). This caused early `apk add` and `apt-get install` attempts to fail with "not found" errors. Fixed this by designing a custom multi-stage Dockerfile that fetches `apk-tools-static` from an active Alpine base and injects it into the distroless n8n image. We re-initialized the database using `--initdb` allowing native Alpine distributions of `py3-pandas` and `py3-requests` to land safely without exceeding rendering timeouts due to compilation.
